为什么 ESKF 中误差状态均值没有长期意义？

前言

在学习 ESKF（Error-State Kalman Filter，误差状态卡尔曼滤波）时，经常会看到这样一句话：

由于 ESKF 的误差状态在每次更新以后会被重置，因此运动方程的均值部分没有太大意义，而方差部分则决定了误差估计的分布情况。

很多人第一次看到这句话时会产生疑问：

  • 卡尔曼滤波不是一直在维护均值和协方差吗？
  • 为什么到了 ESKF，均值突然不重要了？
  • 难道误差状态的均值完全没有意义吗？

事实上，这句话容易引起误解。

更准确的表述应该是：

误差状态均值不是没有意义，而是不具有长期意义。

ESKF 每次更新后都会将误差注入名义状态，并将误差状态重新置零，因此误差均值不会长期积累；真正被长期传播和利用的是误差协方差。

本文尝试从 KF → ESKF 的角度建立直觉。

1. 普通 Kalman Filter 在维护什么？

标准卡尔曼滤波维护一个高斯分布：

x∼N(μ,P) x \sim N(\mu,P) x∼N(μ,P)

其中：

  • μ\muμ：状态均值
  • PPP：状态协方差

例如：

x=[位置 速度] x= \begin{bmatrix} 位置\ 速度 \end{bmatrix} x=[位置 速度​]

则：

μ=[10 1] \mu= \begin{bmatrix} 10\ 1 \end{bmatrix} μ=[10 1​]

表示：

预测阶段：

μ−=Fμ \mu^- = F\mu μ−=Fμ P−=FPFT+Q P^- = FPF^T+Q P−=FPFT+Q

更新阶段：

μ=μ−+K(z−Hμ−) \mu = \mu^- + K(z-H\mu^-) μ=μ−+K(z−Hμ−) P=(I−KH)P− P = (I-KH)P^- P=(I−KH)P−

这里：

μ \mu μ

和

P P P

都是真正长期维护的对象。

名义状态（Nominal State）

δx \delta x δx

称为：

误差状态（Error State）

4. 为什么误差均值不会长期积累？

这是理解 ESKF 的关键。

假设：

当前时刻：

真实位置：

10.0m 10.0m 10.0m

名义状态：

9.8m 9.8m 9.8m

误差状态：

0.2m 0.2m 0.2m

即：

δp=0.2m \delta p=0.2m δp=0.2m

经过观测更新后：

ESKF 会执行：

x^←x^⊕δx \hat x \leftarrow \hat x \oplus \delta x x^←x^⊕δx

即：

9.8+0.2=10.0 9.8+0.2=10.0 9.8+0.2=10.0

名义状态变成：

10.0m 10.0m 10.0m

然后执行：

δx←0 \delta x \leftarrow 0 δx←0

误差状态重置。

因此：

更新完成后：

E[δx]≈0 E[\delta x] \approx 0 E[δx]≈0

下一时刻：

误差重新开始积累。

因此误差状态更像：

名义状态更像：

补丁文件删除。

5. 误差状态均值真的没用吗？

严格来说：

不是。

误差状态均值仍然参与传播。

例如：

误差动力学：

δxk=Fkδxk−1+wk \delta x_k = F_k \delta x_{k-1} +w_k δxk​=Fk​δxk−1​+wk​

均值传播：

E[δxk]=FkE[δxk−1] E[\delta x_k] = F_kE[\delta x_{k-1}] E[δxk​]=Fk​E[δxk−1​]

协方差传播：

Pk=FkPk−1FkT+Q P_k = F_kP_{k-1}F_k^T+Q Pk​=Fk​Pk−1​FkT​+Q

从数学上看：

均值传播依然存在。

但是：

由于每次更新都会执行：

δx←0 \delta x \leftarrow 0 δx←0

因此：

E[δx]≈0 E[\delta x] \approx 0 E[δx]≈0

始终成立。

所以：

误差均值不会像普通 KF 那样不断积累。

ESKF 中最重要的量其实是：

P P P

因为：

位置和姿态之间可能存在耦合：

Ppθ≠0 P_{p\theta}\neq 0 Ppθ​=0

速度和陀螺Bias之间也可能相关：

Pvbg≠0 P_{vb_g}\neq 0 Pvbg​​=0

7. 为什么 FAST-LIO 更关注 P？

观察 FAST-LIO 的预测部分：

核心计算实际上是：

Pk=FkPk−1FkT+Qk P_k = F_kP_{k-1}F_k^T + Q_k Pk​=Fk​Pk−1​FkT​+Qk​

而不是：

E[δx] E[\delta x] E[δx]

原因就在于：

FAST-LIO 默认：

δx=0 \delta x = 0 δx=0

附近线性化。

真正影响后续更新的是：

P P P

因为：

卡尔曼增益由它决定：

K=PHT(HPHT+R)−1 K = PH^T (HPH^T+R)^{-1} K=PHT(HPHT+R)−1

而：

K K K

又决定：

因此：

FAST-LIO、MSCKF、OpenVINS、IKFoM 等系统都会花大量篇幅推导：

F,G,Q,P F,\quad G,\quad Q,\quad P F,G,Q,P

而很少专门讨论：

E[δx] E[\delta x] E[δx]

因为理论上：

E[δx]≈0 E[\delta x] \approx 0 E[δx]≈0

始终希望成立。

总结

ESKF 并不是不维护误差状态均值。

实际上：

E[δx] E[\delta x] E[δx]

依然存在，也参与传播。

但是由于每次更新后都会执行：

x^←x^⊕δx \hat x \leftarrow \hat x \oplus \delta x x^←x^⊕δx

以及：

δx←0 \delta x \leftarrow 0 δx←0

因此误差均值不会长期积累。

从工程实现角度看：

  • 名义状态负责描述系统当前状态；
  • 误差状态负责提供一次性修正量；
  • 协方差负责描述不确定性传播。

因此在 FAST-LIO、MSCKF、OpenVINS 等 ESKF 系统中，真正被长期传播和重点维护的是：

P \boxed{ P } P​

而不是：

E[δx] \boxed{ E[\delta x] } E[δx]​

这就是“ESKF 中误差状态均值没有长期意义，而协方差才是核心”的真正含义。
