BehaviorTree 核心：Decorator Node 流程影响

真正的核心不是 Action Node，而是 Control Node 的执行规则。
Decorator Node 虽然：
# 不负责组织多个节点
但它：
# 会深度影响执行流程，改变节点语义。

一、Decorator 最常见的 6 类流程影响

类型 核心影响 示例场景
Retry 失败后重试 路径跟踪失败时自动重试
Repeat 成功后重复执行 巡逻任务循环执行指定次数
Timeout 超时中断执行 等待电梯超时触发失败
ForceSuccess / ForceFailure 强制修改返回值 记录日志但不中断主流程
Inverter 反转节点执行结果 将“检测到障碍”转为“无障碍”逻辑
RateController 限制节点 Tick 频率 导航中控制路径重规划速率

三、Decorator 的本质力量

不修改节点代码，却能彻底改变节点行为：

  1. 返回值控制（如 ForceSuccess）。
  2. 生命周期管理（如 Retry、Repeat）。
  3. 时间约束（如 Timeout）。
  4. 执行频率调节（如 RateController）。
  5. 逻辑反转（如 Inverter）。

五、终极总结

Decorator Node 本质是：
“节点行为变换器”
它通过封装非侵入式修改，赋予节点以下能力：

  1. 定义何时结束执行。
  2. 定制失败/成功的后处理逻辑。
  3. 约束执行时间与频率。
  4. 解耦逻辑与流程控制。

这种设计哲学，正是 BehaviorTree.CPP 在机器人领域实现高度灵活性和可复用性的核心基石。
