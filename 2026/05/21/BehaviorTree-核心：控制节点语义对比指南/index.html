BehaviorTree 核心：控制节点语义对比指南

真正的核心不是 Action Node，而是 Control Node 的执行规则

一、Sequence vs Fallback（经典对比）

Sequence（顺序执行）

含义：
一步一步完成任务
逻辑：

A→B→CA \rightarrow B \rightarrow CA→B→C

规则：
一个失败 → 全部失败
适用场景：

  • 导航流程
  • 机器人抓取
  • 工业流水线

Fallback（选择器）

含义：
一个失败就尝试下一个
逻辑：
A OR B OR C
规则：
一个成功 → 立刻停止

示例对比

本质区别：

节点类型 执行目标
Sequence 必须全部成功
Fallback 有一个成功即可

ReactiveSequence 实时条件响应优先

四、Sequence vs PipelineSequence（Nav2 核心）

普通 Sequence

规则：
前一个完成 → 后一个才运行
导航表现：
规划一次 → 一直跟踪旧路径

PipelineSequence

规则：
后续节点 RUNNING 时，前面节点仍可 Tick
导航表现：
边跟踪路径 → 边动态重规划

💡 关键价值：这是 Nav2 实现 动态重规划 的核心机制

五、Fallback vs ReactiveFallback（静态选择 vs 动态切换）

普通 Fallback

行为：
首次选中节点后持续执行，不重新检查高优先级条件

ReactiveFallback

行为：
每次 Tick 重新评估所有子节点优先级

动态响应：
急停信号出现 → 下一次 Tick 立即切换到 EmergencyStop

类型 作用对象 典型节点
Control Node 多个子节点关系 Sequence, Fallback, Parallel
Decorator 单个子节点行为 Retry, Timeout, ForceSuccess

终极对比表：机器人领域核心控制节点

对比维度 节点 A 节点 B 本质差异
任务完成逻辑 Sequence Fallback 全成功 vs 一个成功
失败处理策略 Retry RecoveryNode 硬重试 vs 恢复后重试
条件检查方式 Sequence ReactiveSequence 记忆式 vs 实时重检查
路径规划机制 Sequence PipelineSequence 串行执行 vs 流水线重规划
选择器响应性 Fallback ReactiveFallback 静态选择 vs 动态抢占
作用范围 Control Node Decorator 多节点流程 vs 单节点修饰
结果强制 ForceSuccess ForceFailure 结果重写方向
执行模式 Sequence Parallel 串行 vs 并发

Nav2 的核心思想：反应式任务系统

BehaviorTree.CPP 在机器人领域的强大源于 分层反应式设计：

这不是状态机，而是：
✅ 条件驱动的实时响应系统
✅ 分层恢复的鲁棒任务执行
✅ 数据流隐式传递（通过 Blackboard）

掌握这些控制节点的精确语义差异，才是 BehaviorTree.CPP 实战开发的核心能力。
